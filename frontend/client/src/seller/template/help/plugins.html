Seller rules impose business policies and regulations on commercial transactions. Seller rules operate as part of the shopping cart workflow logic. Your seller account depends on some seller rules to be in place for normal operation. All active seller rules are executed sequentially from top to bottom.
Seller rules will appear ordered the way you have manually ordered them.
Click the "Add" action to add a new rule.
Click a rule to edit it.
Use to reorder a rule by dragging it within its group.
Use to remove a rule by dragging it outside the left edge of the screen.
Address
Address rule qualifies billing or shipping address that buyer submits to a shopping cart when he/she proceeds to checkout. The system requires that seller has exactly two active address rules, one for billing and the other for shipping address. An example of proper use of address rule is using it to restrict shipping to a particular country.
Carrier
Carrier rule defines conditionally available and dynamically priced delivery method as an option in a shopping cart. The system requires that seller has at least one active carrier rule. The system supports multiple active carrier rules. An example of proper use of carrier rule is using it to define a delivery method that becomes available when a certain weight of a shipment is below specified limit.
Currency
Currency rule establishes accepted currency. This rule makes an exception in terms of operation as it operates on both, catalog product unit prices, and the shopping cart workflow logic. The system requires that seller has exactly one active currency rule.
Discount
Discount rule defines conditionally applicable product discounts that are applied to shopping cart products after a buyer proceeds to checkout. The system does not require that seller has active discount rules. The system supports multiple active discount rules. An example of proper use of discount rule is using it to define a product discount that becomes available when a certain quantity of a specific product is purchased.
PayPal
PayPal rule defines PayPal as an acceptable payment method that buyer can use to make a payment on a shopping cart. The system does not require that seller has active PayPal rules. The system supports one active PayPal rule.
Tax
Tax rule defines conditional sales tax. The system does not require that seller has active tax rules. The system supports multiple active tax rules. An example of proper use of tax rule is using it to define a sales tax that is collected from buyers with a billing address in a particular region.
